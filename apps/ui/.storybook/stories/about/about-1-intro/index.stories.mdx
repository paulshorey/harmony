**Pre-define any props, not only styles.** Then add more specific props or styles per instance:  

<Code
  variant="transparentDark"
  code={`// 1. Use the provided named export to pre-define props:
import { withButton } from '@ps/ui/components/focus/Button';
const Button = withButton({
  ...props,
  size: 'sm',
  icon: <FontAwesomeIcon icon={faUser} />,
  ss: 'margin: 0 0.875rem 0.875rem 0;',
  onClick: () => alert('clicked!'),
});
// 2. Use your component. Pass unique props per instance:
<Button loading>
  Loading
</Button>
<Button variant="outlined">
  Outlined
</Button>
<Button variant="text">
  Text
</Button>
<Button round />`} />


**You don't have to use style strings.** That's just for convenience. Use any type supported by <CodeInline variant="text" code={`@emotion/styled`} />.

<Code
  variant="transparentDark"
  code={`<Block ss={(props) => css\` padding: 1rem \${props.theme.sizes.card.paddingX}; \`}>`} />

<!--Or use your own preferred solution such as <CodeInline variant="text" code={`props.style`} />, <CodeInline variant="text" code={`props.className`} />, <CodeInline variant="text" code={`props.css`} /> or <CodeInline variant="text" code={`props.sx`} />. -->
  
This library does not use Emotion's preferred <CodeInline variant="text" code={`props.css`} /> because it's not portable, it needs to be set up by every app. But since the css prop actually gets compiled to a standard className, you can use it in any app. Instead, this library adopted the syntax <CodeInline variant="text" code={`props.ss`} />.

<a href="http://localhost:6006/?path=/docs/core-styled-components--page" target="_top">Read more</a> about those <CodeInline variant="text" code={`ss`} /> props, the way styled-components are implemented in this library, and more advanced tricks for coloring elements' text and background.
